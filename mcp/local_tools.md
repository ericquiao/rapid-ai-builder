# Local Tools

## Purpose
Explain how MCP tools should be used during rapid local development.

## Tool Guidance
- `filesystem`: create and edit project files.
- `terminal`: install dependencies, run dev servers, execute tests, and inspect logs.
- `git`: review local changes if the project is tracked.
- `browser testing`: verify the main user journey in the local app.

## Operating Rules
- Prefer local commands over remote services.
- Use browser testing to confirm the happy path after the build.
- Keep tool usage focused on unblocking the localhost application.
