# Name
`scope_summarization`

## Purpose
Summarize the approved scope into a short, build-ready view that keeps the first iteration narrow.

## Inputs
- Problem statement
- Approved scope notes
- Core workflow
- Constraints or exclusions

## Outputs
- Concise scope summary
- Clear in-scope items
- Clear out-of-scope items
- Immediate next build focus

## When To Use
- After discussion and before scope lock
- When the builder needs a tighter summary before planning
- When scope drift starts appearing in the conversation
