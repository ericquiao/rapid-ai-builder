# UI Design Patterns

## Purpose
Provide minimal layout guidance for clean first-pass interfaces.

## Recommended Patterns
- Use one clear primary screen for the main flow.
- Keep navigation shallow.
- Prefer simple cards, tables, forms, and status banners.
- Use Tailwind utility classes consistently.
- Design for desktop first, then ensure mobile is still usable.
- Show loading, empty, and error states for the main interaction.
- Avoid decorative complexity that slows implementation.

## Default Layout
- Header with app title and primary action.
- Main content area with one focused task.
- Secondary details below or in a side panel if needed.
